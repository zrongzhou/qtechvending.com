# SEO 体检报告 — QtechVending 三语站点（2026-07-23）

> 审计对象：`qtechvending-local/`（Next.js App Router，三语 en/zh/ar）
> 审计人：架构师 高见远（基于源码实地核对，非凭空推断）
> 配套报告：`performance-2026-07-23.md`

---

## 1. 现状总览

站点通过 `src/lib/seo.ts` 的 `generatePageMetadata()` 统一产出 `Metadata`（title / description / canonical / hreflang / OpenGraph / Twitter）。三语路由为 `/en/*`、`/zh/*`、`/ar/*`。已具备：`metadataBase`、产品 `Product` JSON-LD、博客 `Article` JSON-LD、`next.config` 的 301 重定向保留 SEO。

但存在两类高危问题：(1) **canonical / og:url 丢掉 locale 前缀**，指向不存在的无 locale 路由（→ 404），且 hreflang 备用集不含 canonical 自身；(2) **sitemap 仅生成英文**。这两点直接影响三语索引与去重，列为 P0。其余为本地化缺失、表单字段缺失、占位符配置等中等 / 低危项。

---

## 2. 问题（按严重程度分级）

### 🔴 高（P0）

#### P0-1 canonical / og:url 缺 locale 前缀，且 hreflang 集合不含 canonical
- **文件定位 + 行号**：`src/lib/seo.ts`
  - L86：`const pageUrl = path.startsWith('http') ? path : \`${baseUrl}${path.startsWith('/') ? '' : '/'}${path}\`;`
  - L122-130：`alternates: { canonical: pageUrl, languages: { en: \`/en${path}\`, 'zh-CN': \`/zh${path}\`, ar: \`/ar${path}\`, 'x-default': \`/en${path}\` } }`
  - L105：`url: pageUrl`（og:url）
- **根因（各页调用）**：所有页面传入的 `path` **不带 locale**，例如：
  - `src/app/[locale]/about/page.tsx` L15 `path: '/about'`
  - `src/app/[locale]/products/page.tsx` L15 `path: '/products'`
  - `src/app/[locale]/solutions/page.tsx` L14 `path: '/solutions'`
  - `src/app/[locale]/contact/page.tsx` L16 `path: '/contact'`
  - `src/app/[locale]/faq/page.tsx` L15 `path: '/faq'`
  - `src/app/[locale]/blog/page.tsx` L15 `path: '/blog'`
  - `src/app/[locale]/page.tsx` L23 `path: '/'`
  - 产品 / 博客 / 分类详情页同理（`/products/${slug}` 等）
- **问题**：`pageUrl = ${baseUrl}/about` —— **一个不存在的无 locale 路由**（站点实际只有 `/en/about`、`/zh/about`、`/ar/about`），搜索引擎访问 canonical 会 **404**。更严重的是：`languages`（hreflang）里都是带 locale 的 `/en/about`、`/zh/about`、`/ar/about`，但 **canonical 并不在其中**，违反"canonical 必须是 hreflang 集合成员之一"的规范，导致 Google 可能忽略 hreflang、产生重复内容判断混乱。
- **具体建议**：
  1. 给 `generatePageMetadata()` 增加 `locale` 参数（`PageSEOOptions` 增加 `locale: string`）。
  2. 将 `pageUrl` 改为 `const pageUrl = \`${baseUrl}/${locale}${path}\``（path 不带前导 locale 时）。
  3. `canonical` 与 `og:url` 均用该带 locale 的 `pageUrl`；`languages` 中 `en/zh-CN/ar/x-default` 仍按现有规则生成，确保 **canonical 出现在集合内**。
  4. 首页 `path:'/'` → `pageUrl = ${baseUrl}/${locale}/`（或 `${baseUrl}/${locale}`，保持与 sitemap 一致）。

#### P0-2 sitemap.xml 仅生成英文（zh/ar 完全缺失）
- **文件定位 + 行号**：`src/app/sitemap.xml/route.ts`
  - L3：`const LOCALES = ['en'] as const;`
  - L6：`const STATIC_PATHS = ['', '/products', '/blog', '/about', '/contact'];`
  - L75-91：双层循环 `for host → for locale(仅 en) → for path`
- **问题**：三语站点，但 sitemap **只列了 `/en/*`**，搜索引擎**永远收不到 `/zh/*`、`/ar/*`**，中文 / 阿拉伯语页面无法被收录。这是最严重的索引缺口之一。
- **具体建议**：
  1. `const LOCALES = ['en', 'zh', 'ar'] as const;`
  2. 循环内 `url = \`${host}/${locale}${path}\`` 已支持多 locale，改 `LOCALES` 即可自动为每个 path 生成三语文与。
  3. 动态路径（`/category/*`、`/products/*`、`/blog/*`，L64-68）在三语下同样需各生成一条。
  4. 保持 `force-dynamic`（L10）以确保反映最新发布内容，或改 `revalidate`（配合 P0-A 性能优化）。

### 🟠 中（P1）

#### P1-1 `generatePageMetadata` 无 locale 参数，description 永远读 `.en`
- **文件定位 + 行号**：`src/lib/seo.ts`
  - L52 签名无 `locale`；L69-73：`defaultTitle = setting.defaultTitle?.en`、`defaultDescription = setting.defaultDescription?.en`、`keywordList = setting.keywords?.en`
  - 页面调用均未传 locale（见 P0-1 各页 L13-27 等）
- **问题**：站点级默认 **description / keywords 恒取 `.en`**；而各页 `generateMetadata` 传的 `title` 多为硬编码英文（见 P1-2）、`description` 多为 `SITE_CONFIG.defaultDescription`（英文）。结果：**zh / ar 页面的 `<meta description>`、OG 描述、keywords 全是英文**，与页面本地化内容严重不符，损害对应语种的搜索点击率与相关性。
- **具体建议**：
  1. 随 P0-1 一起给 `generatePageMetadata` 加 `locale`。
  2. 默认值改按 locale 取：`setting.defaultDescription?.[locale] || setting.defaultDescription?.en || LEGACY...`。
  3. 各页面 `generateMetadata` 已拿到 `params.locale`，将其传入；对 `description` 未显式传值的页面，使用本地化默认描述。

#### P1-2 静态页标题硬编码英文，未本地化
- **文件定位 + 行号**：
  - `src/app/[locale]/about/page.tsx` L16 `title: 'About Qtech Vending | Smart Vending Machine Manufacturer'`
  - `src/app/[locale]/products/page.tsx` L16 `title: 'Products'`
  - `src/app/[locale]/solutions/page.tsx` L15 `title: 'Solutions'`
  - `src/app/[locale]/contact/page.tsx` L17 `title: 'Contact'`
  - `src/app/[locale]/faq/page.tsx` L16 `title: 'FAQ'`
  - `src/app/[locale]/blog/page.tsx` L16 `title: 'Blog'`
- **问题**：`title` 写死英文，`zh` / `ar` 页面 `<title>` 与 SERP 标题均为英文，未利用已存在的本地化文案（如 About 页 `DEFAULT_SECTIONS` 已有 `{en,zh,ar}` 标题）。
- **具体建议**：
  1. 将 `title` 改为 `I18nString` 或从 `SiteSetting.defaultTitle?.[locale]` / 本地化资源取；至少对每个静态页提供 `{en,zh,ar}` 标题映射。
  2. 详情页（产品 / 博客 / 分类）已用 `localized(post.title, locale)` 取本地化标题（如 `blog/[slug]/page.tsx` L29）——静态页应保持一致做法。

#### P1-3 BlogForm 有 `seoTitle` 无 `seoKeywords`（对比 ProductForm 完整）
- **文件定位 + 行号**：
  - `src/components/admin/BlogForm.tsx`：L38 `seoTitle` state、L61 payload 含 `seoTitle`、L169 仅渲染 `TriTextInput label={t('admin.fieldSeoTitle')}`；**全文无 `seoKeywords` 状态 / 输入 / payload**。
  - 对比 `src/components/admin/ProductForm.tsx`：L50 `seoKeywords` state、L108 payload 含 `seoKeywords`、L305 渲染 `TriListInput label={t('admin.fieldSeoKeywords')}`。
- **问题**：博客创建 / 编辑**没有关键词输入框，提交 payload 也无 `seoKeywords`**，导致 `BlogPost.seoKeywords` 始终为空。下游 `seoKeywordList(post.seoKeywords, locale)`（`blog/[slug]/page.tsx` L33、L38）为空 → 关键词**退化为 `[title]`**，博客 SEO 关键词能力形同虚设。
- **具体建议**：
  1. `BlogForm` 增加 `seoKeywords: I18nStringList | null` state（参照 ProductForm）。
  2. 在 SEO 区块（L165-170）增加 `<TriListInput label={t('admin.fieldSeoKeywords')} value={seoKeywords} onChange={setSeoKeywords} />`。
  3. payload（L52-63）增加 `seoKeywords: seoKeywords ?? null`。
  4. 后台 `PATCH/POST /api/admin/blogs` 与 Prisma 写入需支持 `seoKeywords`（模型已有 `seoKeywords Json?`，见 P1-4）。

#### P1-4 BlogPost 无 `seoDescription` 列（Product 有）
- **文件定位 + 行号**：`prisma/schema.prisma`
  - `model BlogPost`（L60-78）：含 `seoKeywords`（L71）、`seoTitle`（L72），**无 `seoDescription`**。
  - 对比 `model Product`（L32-57）：L48 `seoDescription Json?`。
- **问题**：博客描述无法自定义——`blog/[slug]/page.tsx` L31 用 `localized(post.excerpt, locale)` 作为 `description`，而 excerpt 是列表摘要，未必适合做 SERP 描述；无独立 `seoDescription` 字段。
- **具体建议**：
  1. `prisma/schema.prisma` 的 `BlogPost` 增加 `seoDescription Json?`（与 Product 对齐）。
  2. 生成并应用迁移（`prisma migrate` / `db push`）。
  3. `BlogForm` 增加 `seoDescription` 的 `TriTextArea`（参照 ProductForm 的 `seoDescription` 输入）。
  4. `blog/[slug]/page.tsx` 优先用 `post.seoDescription`，回退 `excerpt`。
  5. ⚠️ 此项需 DB 迁移，建议与 P1-3 一并排期。

### 🟡 低（P2）

#### P2-1 `og:locale` 写死 `en_US`
- **文件定位 + 行号**：`src/lib/seo.ts` L108 `locale: 'en_US'`、L109 `alternateLocale: ['zh_CN', 'ar']`
- **问题**：所有语种页面 OG `locale` 均为 `en_US`，与 `zh`/`ar` 不匹配（应为 `zh_CN` / `ar`）。
- **具体建议**：按 `locale` 映射：`en→en_US`、`zh→zh_CN`、`ar→ar`；`alternateLocale` 为其余两者。

#### P2-2 首页 `<title>` 重复品牌名
- **文件定位 + 行号**：
  - `src/app/[locale]/page.tsx` L24 `title: SITE_CONFIG.defaultTitle`（= `'Qtech — Intelligent Vending & Fresh-Flower Automation Equipment'`）
  - `src/app/layout.tsx` L9 `title.template: '%s | Qtech'`
- **问题**：首页把**含 "Qtech" 的完整默认标题**作为 `title` 传入，再经根布局模板拼接 ` | Qtech`，最终渲染为 **`Qtech — Intelligent Vending & Fresh-Flower Automation Equipment | Qtech`**——品牌名重复；`og:title`（`seo.ts` L79 `displayTitle = \`${title} | ${siteName}\``）同样重复。其它页（about/products 等）因传精简 title，仅得 ` | Qtech` 正确后缀。
- **具体建议**：首页 `generateMetadata` 传 `title: SITE_CONFIG.defaultTitle` 时**不要**再走模板 / `displayTitle` 的二次拼接；或首页直接用精简名（如 `'Qtech'` / 本地化品牌名），由模板补 ` | Qtech`。

#### P2-3 站点级 `seoKeywords`（DB 可编辑）几乎未被使用
- **文件定位 + 行号**：`src/lib/seo.ts` L73 `keywordList = setting.keywords?.en`、L97-100 仅在 `options.keywords` 为空时回退使用
- **问题**：各页面均显式传 `buildStaticPageKeywords(...)`（`src/lib/seo-keywords`），导致 `SiteSetting.keywords`（后台可编辑的站点级关键词）几乎从不被采用，后台配置形同虚设。
- **具体建议**：当页面未传 `keywords` 时，将站点级 `keywords` 作为兜底；或在 `buildStaticPageKeywords` 中并入站点级关键词。属低优先级增强。

#### P2-4 分类页缺 JSON-LD
- **文件定位 + 行号**：`src/app/[locale]/category/[slug]/page.tsx`
  - L17-28 `generateMetadata` 已有 SEO 元数据；但**渲染体 L43-65 无任何 `<JsonLd>`**（产品页有 `Product` JSON-LD，博客页有 `Article` JSON-LD，分类页无 `Collection`/`Category` 结构化数据）。
- **问题**：分类列表页缺少 `CollectionPage` / `Category` JSON-LD，损失一条重要的结构化收录信号。
- **具体建议**：在分类页渲染时加入 `<JsonLd data={{ '@type': 'CollectionPage', name, url, ... }}>`（复用 `src/components/JsonLd.tsx` 与 `seo.ts` 既有模式）。

#### P2-5 `/en/qa-*` QA 页无 metadata 且可被索引
- **文件定位 + 行号**：
  - `src/app/robots.ts` L11 `disallow: ['/xiaozhouBackend/', '/api/']`（**未**包含 QA 页）
  - `src/app/[locale]/qa-products/page.tsx`（及 `qa-categories`、`qa-v47-fireworks`、`qa-product-tab`、`qa-products-grid`）：均为 `'use client'` 组件，**无 `generateMetadata` 导出**，继承根布局默认 metadata（标题为占位默认），且**可被搜索引擎索引**。
- **问题**：这些是**测试 / QA 用页面**（代码注释明确 "QA HARNESS — DO NOT SHIP"），却以真实可索引 URL 暴露，会造成重复内容、稀释权重、泄露内部测试页。
- **具体建议（任选）**：
  1. `robots.ts` 增加 `disallow: ['/en/qa-*', '/zh/qa-*', '/ar/qa-*']`（或统一前缀 `/qa-`）；
  2. 更彻底：从生产构建中移除 QA 页，或对其加 `export const metadata = { robots: { index: false, follow: false } }`（需为 Server Component 或 `generateMetadata`）。

#### P2-6 搜索引擎验证码仍是占位符
- **文件定位 + 行号**：`src/app/layout.tsx` L18-25 `verification: { google: 'YOUR_GOOGLE_VERIFICATION_CODE_HERE', other: { 'msvalidate.YOUR_BING_CODE': 'YOUR_BING_VERIFICATION_CODE_HERE', 'yandex-verification': 'YOUR_YANDEX_VERIFICATION_CODE_HERE', 'baidu-site-verification': 'YOUR_BAIDU_VERIFICATION_CODE_HERE' } }`
- **问题**：Google / Bing / Yandex / 百度 验证码均为占位符，搜索引擎站长工具**无法验证站点归属**，影响 Search Console 数据接入与部分增强功能。
- **具体建议**：部署前填入真实验证码（建议改为从环境变量读取，避免硬编码进仓库）；此项为上线前置条件。
